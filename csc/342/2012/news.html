2012.02.03

  1. Re Submissions 04/05, note that if you already have some "packet files" in your directory when you start a run, at least some previously created packet files will likely be overwritten and you will wind up with a mixed-up bunch of packet files that cannot be reassembled after a new run, so you should always start a new run with a "clean" directory (no packet files in it). Also note that though the "theoretical" message-size limit is 5000 "words" for your packet files (1000 files with five words each), a more practical and "safer" limit is 1000 words, since (again "theoretically") each packet file might only contain a single word (however unlikely this might be). [Thanks to Evan Lowry for some comments leading to these comments of mine.]

2012.01.19

  1. Thanks to Evan Lowry for pointing out the following two problems with Submission 02 (and they would also have appeared in Submission 03, but they have been rectified and new executables have been installed for both Submissions, as of Thu Jan 19 13:46:21 AST 2012):
    • If you entered the name of a file that didn't exist when reading in an island, then after you got the message about the file not existing a second message was shown--the same message that you got if you tried to do something with an island that didn't exist. That was at best a redundant, and at worst a possibly confusing, message and it has been eliminated. Bottom line: Now you go directly back to the menu after getting the message that the file does not exist.
    • Second, if you already had an island read in and tried to read in a new one, the current island was wiped out whether you succeeded in reading in a new one or not. Now, if the the new file doesn't exist, you retain the old island.
