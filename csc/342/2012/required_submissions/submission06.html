Overview

This week your task is to gain a reasonable comfort level working with binary files, and in particular the "random access" aspect of working with binary files. To do that you will write a menu-driven program that allows the user to create a binary file of real numbers and then read from and/or write to that file at various locations within the file. The program also permits the user to display a "status report" after any particular operation.

Steps to Perform

  1. Download the sample executable and spend some time "putting it through its paces". Be sure to display the "status report" frequently and reconcile what you have just done with the given output.
  2. Now download the supplied .hpp file and note the two functions it contains. Having these functions will save you some "busy work". Use this file as a #include file.
  3. When you feel comfortable that you understand all aspects of what the program is doing, write pseudocode to describe the program's behavior.
  4. Translate your pseudocode (incrementally) into C++ and test (incrementally) until you have the complete program working in the same way as the sample executable.
  5. When you are finished, make sure your source code file is identified, formatted, and documented properly.
  6. Finally, submit the required files over the web.

Additional Notes, Requirements, Specifications and/or Hints (if any)

  1. This week, perhaps for the first and last time ever, you are permitted to write your program as a single (large) main function. It will be essentially a single do...while loop containing a fairly large switch statement to handle all the menu options. By not requiring you to break up main() into several smaller functions we let you put all your time into dealing with binary file matters.
