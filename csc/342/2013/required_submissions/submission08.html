Overview

This week you will design and write a program that can evaluate either prefix or postfix expressions read in from the command line. Each such expression must have positive integer operands and involve only the (binary) operators +, -, * and/or /. Some error checking is also required on this occasion.

Steps to Perform

  1. Copy the sample executable demo_eval_pre_post.exe and run it several times with different (valid) input. Below are several suggestions of typical valid examples showing how your program must work. Try them (and lots of others of your own choosing) with the sample executable: 
    prompt> demo_eval_pre_post 3 4 5 + *
3 4 5 + * = 27
prompt> demo_eval_pre_post 2 3 * 15 5 7 + / +
2 3 * 15 5 7 + / + = 7
prompt> demo_eval_pre_post / * + 6 2 3 - 9 7
/ * + 6 2 3 - 9 7 = 12
prompt> demo_eval_pre_post + * 2 3 / 15 + 5 7
+ * 2 3 / 15 + 5 7 = 7
  2. Then, to generate the kinds of errors which the program must recognize and report, try the following examples: 
    prompt> demo_eval_pre_post + * 2 3
prompt> demo_eval_pre_post + x 2 3
prompt> demo_eval_pre_post + * 2 3 4 5
prompt> demo_eval_pre_post 3 3 * 5
prompt> demo_eval_pre_post 2 3 4 + x
prompt> demo_eval_pre_post 2 3 + *
prompt> demo_eval_pre_post 2 3 4 5 6 + *
    You should make up and try a lot more of these kinds of invalid examples to get a good feel for the error-reporting performed by the program.
  3. Write pseudocode for a solution of this problem and revise it until you are convinced that a program that implements your pseudocode will be the one you want.
  4. Translate your pseudocode into C++ and continue testing your program until you are convinced it works correctly.
  5. Make sure your source code is identified, formatted, and documented properly, according to our current rules and guidelines.
  6. Finally, submit the required files over the web.

Additional Notes, Requirements, Specifications and/or Hints (if any)

  1. You may find it useful to note that if you are dealing with a prefix expression, the first "token" in the input will be an operator, while if you are dealing with a postfix expression, the first token will be an operand.
  2. We discussed in class the algorithms for evaluating both prefix and postfix arithmetic expressions, so all you have to do to get this program to evaluate such expressions is implement those algorithms. Thus the interesting part of this submission is making your program recognize and report some potential errors. You may do this however you like, but here is a very simple exception class that you may place in your program and then throw (and catch) appropriate Exception objects at the appropriate places in your program: 
    class Exception
{
public:
    Exception(const string& message) {this->message = message;}
    string getMessage() {return message;}
private:
    string message;
};
    Note that unlike in Java, in C++ exception classes do not have to inherit from any particular C++ classes, exception or otherwise. Also, there is no throws keyword in C++. Among other things, this means that in C++, functions that may throw an exception but not catch it themselves are not required to so indicate with a "throws clause" in the function header.
  3. A good approach to this submission, as is often the case but here especially, will be to get both types of evaluation working properly before trying to deal with error handling.
