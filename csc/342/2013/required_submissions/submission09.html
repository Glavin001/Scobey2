Overview

This week you will design and write a program that allows the user to create and display balanced binary search trees of characters. Since we are specifying the type, we are not going to use a template BST class. In fact, we are not using a BST class at all, just old-fashioned free functions to do all the work. You will have enough to keep you busy, probably, without dealing with templates or even "ordinary" class infrastructure.

We shall only deal with "small" trees (height 3 or less, and therefore 15 nodes or fewer).

Users may enter their own character values, or generate them randomly. The values are first placed into a vector, from which the balanced binary search tree may be generated.

The tree is displayed as an actual tree shape.

In the past you have often been given a .h specification file and been required to implement the specifications in that file. This time you are given much more freedom ... to first decide what functions you need, then to put their specifications in your own header file (balanced_bst.h), and their implementations in a separately-compilable implemenation file (balanced_bst.cpp).

Steps to Perform

  1. Copy the sample executable and run it. Notice there is no "Get information" option, since none is really needed. The menu options themselves, together with the discussion in this document, give you all you need to know.
  2. When you understand what is required, write pseudocode for a program to emulate this executable and revise it until you are convinced that an implementation of your pseudocode will solve the problem.
  3. Translate your pseudocode into C++ and continue testing your program until you are convinced it works correctly.
  4. Make sure all of your source code files are identified, formatted, and documented properly, according to our current rules and guidelines.
  5. Finally, submit the required files over the web.
